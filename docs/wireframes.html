Design Notes

  • Minimal form — email + password only (no OAuth for MVP)
  • Backend returns field-level validation errors displayed inline below each input
  • On success: store access token in memory (Zustand), refresh token set via HttpOnly cookie by backend
  • "Forgot password?" is out of MVP scope — link present but shows "coming soon" toast
  • If already logged in (valid refresh token), auto-redirect to /dashboard

Design Notes

  • Shared component: HabitFormView renders in create mode (/habits/new) or edit mode (/habits/:id/edit)
  • Edit mode: title becomes "Edit Habit", button becomes "Save Changes", fields pre-filled from existing habit
  • Frequency selector: "Daily" / "Weekdays" are toggle buttons; "Custom" reveals the day-of-week picker below
  • Custom days: tap to toggle each day; at least 1 day must be selected; streak logic uses this schedule
  • Colour picker: 8 preset colours (no free-form picker — keeps UI simple, avoids ugly combos)
  • Selected colour gets a white border ring; default is green (first option)
  • Client validation (React Hook Form + Zod): name required + max 100 chars, frequency required, ≥ 1 day if custom
  • Cancel navigates back (history.back()); no unsaved-changes prompt for MVP
  • On success: redirect to /dashboard with a success toast

Design Notes

  • Top 3 get podium-style display; ranks 4–10 in a list; current user is shown at bottom in the opted-in state illustrated here
  • Backend: GET /api/v1/stats/leaderboard → Redis ZREVRANGE leaderboard:total_checkins 0 9 WITHSCORES
  • User's own rank: ZREVRANK leaderboard:total_checkins {userId}
  • "7 check-ins away from #13" — motivational nudge calculated from score difference with next rank
  • Leaderboard updates on every check-in via ZINCRBY (O(log N))
  • Privacy: only display name shown, no email; only users who have enabled "Show on leaderboard" appear here

Design Notes

  • Timezone dropdown: searchable list of IANA timezones; auto-detected on registration but changeable here
  • Timezone warning is always visible below the dropdown — not a modal, just inline context
  • Toggle switches update immediately (optimistic) via PATCH /api/v1/users/me/preferences
  • Profile save is a separate action (button) since name/timezone changes are more consequential
  • Delete Account: opens a confirmation modal requiring the user to type "DELETE" before proceeding (out of MVP scope — button present but shows "coming soon")
  • Reminder time: 24-hour dropdown in 30-minute increments; default 20:00 (8 PM local)
  • Leaderboard opt-out: when disabled, user's data is excluded from ZSET and not shown to others
